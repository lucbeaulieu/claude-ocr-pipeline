# From Paper to Digital: An Automated Handwritten Notes OCR Pipeline

> **Connecting Claude AI to Apple Notes and Apple Reminders via macOS LaunchAgents**

A fully automated pipeline that takes photos of your handwritten meeting notes, transcribes them with AI (Claude), exports structured Markdown files directly into Apple Notes with full formatting, and sends extracted action items straight into Apple Reminders — all triggered automatically the moment Claude writes a file to a watched folder.

## 1. Overview & Architecture

This pipeline was designed for knowledge workers who take handwritten notes during meetings, seminars, or research sessions and want those notes automatically digitized, organized, and actionable — without any manual copy-paste steps. It combines three technologies:

| Component | Technology | Role |
|---|---|---|
| AI Transcription | Claude AI (Anthropic) | OCR + task extraction |
| Notes Import | macOS FSEvents + Swift | Watch folder → Apple Notes |
| Task Import | macOS FSEvents + Swift | Watch folder → Apple Reminders |
| Persistence | LaunchAgent (launchd) | Always-on background daemons |

### 1.1 Pipeline Flow

```
  Handwritten PDF (photo / scan)
        │
        ▼  Uploaded to Claude AI
  ┌─────────────────────────────────┐
  │  Claude OCR Skill               │
  │  Step 1-3 : Transcription + Tags│
  │  Step 4   : Assemble .md file   │
  │  Step 4b  : Extract tasks .json │
  │  Step 5   : Weekly summary .md  │
  └──────────┬──────────────────────┘
             │
    ┌────────┴────────┐
    ▼                 ▼
~/Documents/     ~/Documents/
Claude-OCR-      Claude-OCR-
Notes/*.md       Tasks/*.json
    │                 │
    ▼ (LaunchAgent)   ▼ (LaunchAgent)
md2notes_watcher  tasks2reminders_watcher
    │                 │
    ▼                 ▼
Apple Notes       Apple Reminders
(Handwritten       (Inbox list)
 Notes folder)
    │                 │
    ▼                 ▼
.md deleted       .json deleted
after validation  after validation
```

### 1.2 Key Design Principles

- **Zero manual steps:** once installed, drop a PDF into Claude and everything flows automatically.
- **Validated deletion:** watch folder files are only removed after confirmed successful import into Apple Notes / Reminders.
- **Dual redundancy:** FSEvents callback + 5-second polling timer ensures no file is ever missed.
- **Duplicate prevention:** `NSLock` mutex guards prevent FSEvents and polling from processing the same file twice.
- **Graceful failure:** on any error the file is preserved and the error logged — nothing is lost silently.
- **Privacy-first:** all processing stays on your Mac; no data leaves your machine after Claude writes the files.

## 4. Claude Skill Setup

The Claude skill teaches Claude how to transcribe handwritten notes, extract tasks, and write output files to the correct watched folders. It must be installed before using the pipeline.

### 4.1 Install the Skill

1. Open the Claude desktop app.
2. Click your profile icon → **Settings** → **Skills**.
3. Click **Add Skill Folder** and navigate to a folder of your choice (e.g., `~/Documents/ClaudeSkills/`).
4. Inside that folder, create a subfolder named `handwritten-notes-ocr`.
5. Copy `skill/SKILL.md` from this repository into that subfolder — the filename must be `SKILL.md`.
6. Claude will detect the skill automatically on the next conversation.

### 4.2 What the Skill Does

When you upload a handwritten PDF to Claude, the skill orchestrates a multi-step workflow:

- **Steps 1–3:** Parse filename metadata, perform vision-based OCR transcription, apply domain auto-tags (`#research`, `#grant`, `#student`, `#clinical`, `#admin`, etc.)
- **Step 4:** Assemble a structured Markdown file and write it to `~/Documents/Claude-OCR-Notes/`
- **Step 4b:** Extract all action items into a JSON task file and write it to `~/Documents/Claude-OCR-Tasks/`
- **Step 5:** Generate an executive weekly summary when 3+ files from the same calendar week are processed.

### 4.3 Filename Convention

For best results, name your uploaded PDFs using this convention:

```
YYYYMMDD_HHMMdescriptor.pdf

Example: 20260317_1030research-meeting.pdf
```

Claude will extract the date, time, and descriptor to name output files correctly.

### 4.4 Domain Auto-Tags

| Tag | Trigger Keywords |
|---|---|
| `#research` | experiments, results, data, analysis, publication, manuscript |
| `#grant` | NSERC, CIHR, budget, funding, FCI, deadline, proposal |
| `#student` | MSc, PhD, postdoc, HQP, mentoring, thesis, defence |
| `#clinical` | patient, treatment, dose, linac, brachytherapy, HDR, LDR, RT |
| `#dosimetry` | scintillator, detector, dosimeter, calibration, QA |
| `#admin` | committee, agenda, minutes, policy, budget, report |
| `#governance` | senate, faculty council, board, dean, rector, accreditation |
| `#teaching` | course, lecture, syllabus, exam, grading, curriculum |
| `#industry` | partner, contract, NDA, IP, license, company, startup |
| `#conference` | COMP, AAPM, ESTRO, abstract, invited talk, congress |
| `#action-required` | TODO, ⚠, !, action, follow-up, deadline, urgent |

## 5. Installation

### 5.1 Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/claude-ocr-pipeline.git
cd claude-ocr-pipeline
```

### 5.2 One-Command Install

Place all repository files and run:

```bash
bash install_pipeline.sh
```

The script performs these steps automatically:

1. Creates `~/Documents/Claude-OCR-Notes/` and `~/Documents/Claude-OCR-Tasks/`
2. Creates `~/Library/Scripts/` and `~/Library/LaunchAgents/` if they do not exist
3. Copies all `.py` and `.sh` scripts to `~/Library/Scripts/`
4. Sets executable permissions on shell scripts
5. Compiles both Swift watchers to native binaries (~30–60 seconds each)
6. Writes personalized `.plist` files (auto-detects your username)
7. Loads both LaunchAgents via `launchctl`
8. Verifies both agents are running and prints a confirmation

### 5.3 Grant macOS Permissions (one-time)

After installation, trigger the Reminders permission dialog by running this once in Terminal:

```bash
osascript -e 'tell application "Reminders" to return name of every list'
```

macOS will display a permission dialog — click **OK**. This step is required only once.

> **Why Not Folder Actions?**
> macOS Folder Actions are blocked from accessing `~/Documents` by the TCC (Transparency,
> Consent & Control) privacy framework since macOS 13. This pipeline uses LaunchAgents
> instead, which run in your user session with your full permissions — no TCC issues,
> no silent failures.

## 6. Apple Notes Pipeline

### 6.1 How It Works

The `md2notes_watcher` binary monitors `~/Documents/Claude-OCR-Notes/` using macOS FSEvents with a 5-second polling safety net and a `NSLock` mutex to prevent duplicate processing. When a new `.md` file appears, `md2notes_convert.py`:

1. Reads the Markdown file line by line
2. Converts Markdown syntax to Apple Notes-compatible HTML (headings, bold, lists, tables, blockquotes, code blocks)
3. Calls Notes via AppleScript to create or update the note in a folder named **Handwritten Notes**
4. Verifies the note exists in Notes before deleting the `.md` source file

### 6.2 Markdown → HTML Conversion

| Markdown | Result in Apple Notes |
|---|---|
| `# H1` / `## H2` / `### H3` | Styled HTML headings with size hierarchy |
| `**bold**` / `__bold__` | `<strong>` formatting |
| `*italic*` | `<em>` formatting |
| `~~strikethrough~~` | `<s>` formatting |
| `- item` / `* item` | HTML unordered list |
| `\| col \| col \|` | HTML table with borders |
| `> blockquote` | Styled blockquote (used for `#tags`) |
| ` ```code``` ` | Monospace `<pre>` block with background |
| `---` | Horizontal rule divider |

### 6.3 Notes Folder Structure

All notes are created inside a dedicated folder named **Handwritten Notes** in Apple Notes. If this folder does not exist, it is created automatically on first import. Notes are named using the filename convention: `YYYY-MM-DD_descriptor`.

## 7. Apple Reminders Pipeline

### 7.1 How It Works

The `tasks2reminders_watcher` binary monitors `~/Documents/Claude-OCR-Tasks/` using FSEvents with a 5-second polling safety net. When a new `.json` file appears, `tasks2reminders_convert.py`:

1. Reads the JSON task array
2. Activates Reminders app and waits 2 seconds for it to be ready
3. For each task: creates a reminder in the **Inbox** list with title, due date, and contextual note
4. Due dates use numeric AppleScript components (locale-independent — works in FR, EN, and all system languages)
5. Verifies each reminder exists before proceeding to the next
6. Deletes the `.json` file only after all reminders are validated

### 7.2 Task JSON Schema

Claude writes one JSON file per transcribed document. Each task object contains:

```json
{
  "source":    "YYYY-MM-DD_descriptor",
  "generated": "YYYY-MM-DD",
  "tasks": [
    {
      "title":  "Action verb + concise description (max 80 chars)",
      "due":    "YYYY-MM-DD or empty string",
      "owner":  "Person responsible or empty string",
      "note":   "Brief context — source section, rationale (max 120 chars)"
    }
  ]
}
```

### 7.3 Task Detection Signals

Claude detects action items using these signals during transcription:

- **Explicit tags:** any item tagged `#action-required`
- **Prefix symbols:** `→`, `⇒`, `!`, `⚠`, `- [ ]` (checkbox)
- **Deadline language:** date, échéance, avant, d'ici, retour le, due
- **Action verbs:** transférer, confirmer, soumettre, vérifier, envoyer, préparer, valider, décider
- **Named assignments:** items attributed to a person with a pending action

## 8. Monitoring & Troubleshooting

### 8.1 Log Files

| Log File | Contents |
|---|---|
| `~/Library/Logs/md2notes_watcher.log` | Notes import events, OK/DONE confirmations |
| `~/Library/Logs/md2notes_watcher_error.log` | Conversion errors, AppleScript failures |
| `~/Library/Logs/tasks2reminders_watcher.log` | Reminders creation events, OK/DONE confirmations |
| `~/Library/Logs/tasks2reminders_watcher_error.log` | JSON errors, Reminders API failures |

Monitor in real time:

```bash
# Watch Notes pipeline
tail -f ~/Library/Logs/md2notes_watcher.log

# Watch Reminders pipeline
tail -f ~/Library/Logs/tasks2reminders_watcher.log
```

### 8.2 Check Agent Status

```bash
# Both agents should show a PID (non-zero first column)
launchctl list | grep -E 'md2notes|tasks2reminders'

# Expected output:
# 12345   0   com.YOURNAME.md2notes
# 12346   0   com.YOURNAME.tasks2reminders
```

### 8.3 Restart Agents

```bash
USERNAME=$(whoami)
launchctl unload ~/Library/LaunchAgents/com.${USERNAME}.md2notes.plist
launchctl load   ~/Library/LaunchAgents/com.${USERNAME}.md2notes.plist

launchctl unload ~/Library/LaunchAgents/com.${USERNAME}.tasks2reminders.plist
launchctl load   ~/Library/LaunchAgents/com.${USERNAME}.tasks2reminders.plist
```

### 8.4 Test the Pipeline Manually

```bash
# Test Notes pipeline
echo "# Test Note" > ~/Documents/Claude-OCR-Notes/test.md
sleep 5
# Check Apple Notes for a new note named 'test' in Handwritten Notes

# Test Reminders pipeline
cat > ~/Documents/Claude-OCR-Tasks/test.json << 'EOF'
{ "source": "test", "generated": "2026-01-01",
  "tasks": [{"title": "Test reminder", "due": "", "owner": "", "note": "Pipeline test"}] }
EOF
sleep 5
# Check Apple Reminders Inbox for 'Test reminder'
```

### 8.5 Common Issues

| Symptom | Solution |
|---|---|
| Files not disappearing from watch folder | Check error log; restart the relevant LaunchAgent |
| Reminders permission error (-1728) | Run the `osascript` permission command (Section 5.3) |
| Swift compile fails | Run: `xcode-select --install` |
| LaunchAgent not loading | Check plist path: `ls ~/Library/LaunchAgents/` |
| Notes created without formatting | Ensure `md2notes_convert.py` is present in `~/Library/Scripts/` |
| Agent shows exit 1 repeatedly | Check `_error.log` for the specific Python traceback |
| Duplicate reminders created | Recompile watchers — mutex fix requires latest `.swift` sources |

## 9. Uninstall

```bash
USERNAME=$(whoami)

# Unload agents
launchctl unload ~/Library/LaunchAgents/com.${USERNAME}.md2notes.plist 2>/dev/null
launchctl unload ~/Library/LaunchAgents/com.${USERNAME}.tasks2reminders.plist 2>/dev/null

# Remove files
rm -f ~/Library/LaunchAgents/com.${USERNAME}.md2notes.plist
rm -f ~/Library/LaunchAgents/com.${USERNAME}.tasks2reminders.plist
rm -f ~/Library/Scripts/md2notes_watcher
rm -f ~/Library/Scripts/md2notes_watcher.swift
rm -f ~/Library/Scripts/md2notes_convert.py
rm -f ~/Library/Scripts/md2notes_convert.sh
rm -f ~/Library/Scripts/tasks2reminders_watcher
rm -f ~/Library/Scripts/tasks2reminders_watcher.swift
rm -f ~/Library/Scripts/tasks2reminders_convert.py
rm -f ~/Library/Scripts/tasks2reminders_convert.sh

# Optionally remove watch folders (data is preserved in Notes/Reminders)
# rm -rf ~/Documents/Claude-OCR-Notes
# rm -rf ~/Documents/Claude-OCR-Tasks
```
